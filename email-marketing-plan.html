Sender.net Setup

Create these custom fields and tags in Sender.net before building any automation. All 8 campaigns depend on this foundation. Your app writes to these fields via the Sender.net API the moment a subscriber completes the assessment.

Custom Fields — Subscribers → Custom Fields

Field nameTypeValues
archetype_primaryTextThe Director, The Guardian, etc.
archetype_secondaryTextSame archetype options
journey_stageTextprospect, active, deck-owner, subscriber, member
deck_purchasedDateSet via Zapier on Shopify purchase
subscription_tierTextfree, sensei, journal-pro, annual
journal_entry_countNumberUpdated from app via API
last_loginDateUpdated on each app login

Tags — Applied by App or Zapier

assessment-complete
deck-purchased
sensei-limit-reached
journal-active (5+ entries)
dormant-14 (no login 14 days)
skool-member
course-enrolled-[archetype]
course-[archetype]-available
annual-member
unengaged-pre-assessment

API Integration Point

Your app calls the Sender.net API immediately after assessment completion — before showing the result page. This ensures Campaign 1 fires even if the user closes the browser after seeing their archetype.

End-to-end

How the campaigns connect

Every subscriber enters at Campaign 8 (if no archetype) or Campaign 1 (if archetype is known). All other campaigns are triggered by behavior. No manual sends required.

Entry
  • New subscriber → Campaign 8 (onboarding)
  • Takes assessment → archetype set
  • Campaign 1 fires automatically
  • App writes archetype to Sender.net via API
Conversion triggers
  • Buys deck → Campaign 2 (post-purchase)
  • Hits Sensei limit → Campaign 3 (upgrade)
  • 5 journal entries → Campaign 4 (Journal Pro)
  • Course launches → Campaign 6 (archetype-targeted)
Retention + LTV
  • No login 14 days → Campaign 5 (win-back)
  • 60 days as subscriber → Campaign 7 (annual)
  • Annual purchased → full Skool + bundle access
  • Unengaged → suppressed, list stays healthy
